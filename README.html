RCXDatalog

Introduction

RCXDatalog is my first attempt to write a GUI-based Unix tool to interact with the Lego Mindstorms RCX brick. Using RCXDatalog, you can upload the datalog of the RCX and display it on your screen or print it to a PostScript printer. Also, in the source code you will find some additional widgets to show light and touch sensor readings of the RCX. Presently, these widgets are not used in the main program.

RCXDatalog is inspired by Mark Overmars's RCX Command Center, which runs in Windows. The communication code (send.cpp and send.h) is based on the program "send.c" by Kekoa Proudfoot. You can find it at http://graphics.stanford.edu/~kekoa/rcx/tools.html.

  • Requires Qt2.x, the Qwt widget library and tmake.
  • Tested on Linux 2.2.17 (SuSE 7.0), with the RCX1.0.

Installing RCXDatalog

  • Make sure that the following libraries and tools are installed on your system.
    • Qt Library. Available from Troll Tech AS: http://www.troll.no
    • The Qwt widget Library. Available from http://sourceforge.net/projects/qwt
    • tmake. Available from Troll Tech AS: ftp://ftp.troll.no/freebies/tmake
  • If you like, edit the file "send.h" to set DEFAULTTY to your favourite serial port. You can also set the environment variable RCXTTY.
  • Edit the file "rcxdatalog.pro", so that the INCLUDEPATH includes the location of the Qwt include files.
  • Type: tmake rcxdatalog.pro > Makefile
  • Type: make
If everything goes all right, the program will be compiled now.

If you have any installation problems, please let me know at roel@science.uva.nl. If you have any comments, additions, tips or if you have succesfully installed RCXDatalog, I would appreciate it if you would leave me a message at roel@science.uva.nl.

Using RCXDatalog

Using RCXDatalog is quite straight forward. Start RCXDatalog by typing
  • rcxdatalog
on the command prompt.

If the infra red tower is connected to the correct serial port (set the environment variable RCXTTY if necessary), and your RCX is turned on and within reach of the infrared tower, the main window will appear on your screen.

If anything goes wrong at this stage, you will be warned by a dialog window. If things go wrong at a very premature stage (for example, if the serial port cannot be found), you may be warned on the terminal from which you have started RCXDatalog.

Now, make sure that the datalog of your RCX contains some data, for example a number of light sensor readings. Then press Upload datalog. After a while a graph of the datalog values will appear in the main window.

You can print the graph using File->Print. Exit using File->Quit.
